Markdown like syntax for Edward Tufte style document

We started working on a documentation engine to create documents such as printed user manuals, getting started guides, online help, training handouts and internal documents at Foresptin.

Initially we were working with editors such as Microsoft Word. It was alright when we had a couple of documents but it wasn't easy to maintain formats and manage them as larger documents came in. Then we started looking at alternatives such as LaTeX and Markdown.

Both these options would make managing the documentations much easier. The documents could be version controlled with git, which lets us do a number things such as branching, etc. Although LeTeX gives a lot of flexibility, it was a little too complicated, especially for non-technical writers. Markdown on the other hand was much simpler but didn't support some of the key features we wanted. Markdown doesn't work well when printing. It breaks pages random places. The other problem was that Markdown didn't support sidenotes.

R Markdown suported formatting inspired by Tufte. This was the best already available option for us. We honestly didn't have very good reasons not to use it and develop our own.

However there were a few advantages of developing our own tool:

  • Give us total control

    We would be able to modify the tool to perfectly fit our needs

  • Mardown doesn't have a structure

    For instance, in the following document,

    #Heading1
Intro
#Heading2
Paragraph
#Heading3
Paragraph
Conclusion

    it is not clear whether Conclusion belongs to Heading1 or Heading3. This again gives some trouble when paginating.

  • R Markdown seemed to be use verbose syntaxes for some of the commonly needed functions

DocScript Project

Docscript is available on Github. It is not fully baked yet but you can give it a try.

Here are some of the sample documents we've created:

  • Benford's Law Test
  • Forestpin Dashboard
  • Correlation Test

Design

Docscript uses indentation to specify hierarchy of content.

Example

###Heading1

 Introduction

 * Point one

  Description about point one

 ####Subtopic

  Subtopic content

  More subtopic content

 This belongs to Heading1

Althought indentation doesn't help much in standard rendering except in a a few cases such as lists it's has a number of of other uses. It lets us programmetically set page breaks when printing (This is not implemented yet). Other advantage is we can use code folding when editing; which come handy when working with large documents.

Another key feature of docscript is sidenotes. You can have notes as well as images in sidenotes.

We've also changed some syntaxes of Markdown, such as << and >> for links instead of []() of markdown, because we felt it was a little more intuitive (resemblence with HTML tags).

Docscript also introduces HTML blocks marked by `<<<`, where you can add any HTML content.

Reference

Headings

This is very much like Markdown. # for level 1 headings `## for level 2 headings and so on. The content that belongs to the heading should indented.

Paragraphs

Again similar to Markdown. Paragraphs are separated by blank lines.

List

Ordered lists begin with - while unordered lists begin with * . Since lists are also structured with indentations, it's easy to have lists within lists.

- Introduction
- Analyses

 - Daily analysis
 - Benford's law
 - Timeseries analysis
- Visualizations

 * Treemap
 * Bar charts
 * Dashboard

Media

Images can be added with !.

Special Blocks

This is similar to block quotes.

+++
 This is a special segment.

 Can have all the other things like images.

 !assets/blur.png
+++

This is a special segment.

Can have all the other things like images.

Code Blocks

Code blocks are identified by three backtick quotes (`). Three ` should be placed at the beginning and end of code block.

Html Blocks

Html blocks are identified by <<< at the begining and end of html blocks.

Bold

Text can be made bold with **.

**This is bold** and this is not.

**This entire paragraph is bold.
Still the same paragraph.

Italics

Text can be made italics with --.

**This is italics** and this is not

**This entire paragraph is in italics.
Still the same paragraph
