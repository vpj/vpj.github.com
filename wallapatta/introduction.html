Wallapatta

Wallapatta has a syntax similar to Markdown and uses a layout inspired by handouts of Edward R. Tufte1. We are using it to generate software user manuals at Forestpin.

Star

Overview

Wallapatta uses indentation to specify hierarchy of content.

Example

###Heading1

 Introduction

 * Point one

  Description about point one

 ####Subtopic

  Subtopic content

  More subtopic content

 This belongs to Heading1

Indentation lets you do things like include content within lists. And it helps editing large documents because standard code folding can be used with Wallapatta. Also the hierarchy of content is important for printing - to identify where to break pages.

Another key feature of wallapatta is sidenotes. You can have notes as well as images in sidenotes.

We've changed some syntaxes of Markdown; for instance, << and >> are used for links instead of [](), because we felt it was a little more intuitive (resemblence with HTML tags).

Usage

Online Editor

Use the online editor to try Wallapatta syntax.

Building CLI

The command line interface requires nodejs and coffeescript. Once you clone the Github project, you need to get the git submodules with git submodule init and git submodule update after cloning wallapatta.

cake clean
cake build

The compiled javascript files and other assets will be placed within build directory. index.html file contains the onlide editor.

Compiling a wallapatta document

./wallapatta.coffee -i input_file -o output_file

If the default template is used. the assets in the build directory should be in the same location as the html file. You can specify the template using -h option. The default template is templates/page.coffee
